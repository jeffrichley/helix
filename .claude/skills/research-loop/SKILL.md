Use this skill for any task that involves iterative improvement, investigation, or
experimentation.

## Loop

1. Restate the goal, success metric, constraints, and stop conditions.
2. Inspect the current environment and identify the baseline.
3. Form one bounded hypothesis.
4. Define the smallest meaningful action that tests it.
5. Execute the action through available tools.
6. Evaluate the result against the stated metric or evidence standard.
7. Tag the outcome as `keep`, `discard`, `branch`, `retry`, `escalate`, or `stop`.
8. Record what was learned and choose the next step.

## Preferences

Prefer:

- one major variable at a time
- high-information, low-cost steps
- reversible actions
- explicit baselines
- written evaluation criteria before execution

Avoid:

- changing many things at once unless the domain requires it
- running work that cannot be evaluated
- broad exploratory changes without a recording discipline

## Required Record

For each meaningful step, capture:

- goal
- baseline
- hypothesis
- action
- result
- evaluation
- decision
- next step
